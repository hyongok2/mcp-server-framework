MCP Server Framework 문제 해결 가이드

=== 일반적인 문제들 ===

1. 서버가 시작되지 않는 경우
   - appsettings.json 파일 존재 여부 확인
   - 로그 디렉토리 권한 확인
   - 포트 충돌 확인 (기본값: 5000)

2. 도구가 로드되지 않는 경우
   - tools/ 디렉토리 존재 여부 확인
   - DLL 파일이 Whitelist에 포함되어 있는지 확인
   - McpToolGroupAttribute가 올바르게 설정되어 있는지 확인

3. 프롬프트가 작동하지 않는 경우
   - JSON 파일과 템플릿 파일이 모두 존재하는지 확인
   - templateFile 경로가 올바른지 확인
   - 필수 매개변수가 모두 제공되었는지 확인

4. 리소스에 접근할 수 없는 경우
   - docs/ 디렉토리 존재 여부 확인
   - 파일 확장자가 지원되는 형식인지 확인
   - 파일 권한 확인

=== 로그 확인 방법 ===

로그 파일 위치: C:\Logs\MCPServer\ (기본값)
또는 appsettings.json에서 설정한 디렉토리

중요한 로그 키워드:
- "Loaded tool group" - 도구 로딩 성공
- "Failed to load" - 로딩 실패
- "Validation failed" - 요청 검증 실패
- "Method not found" - 지원하지 않는 메서드

=== 성능 튜닝 ===

1. 로그 레벨 조정
   - 프로덕션: "Info" 또는 "Error"
   - 개발: "Debug"

2. 파일 캐싱
   - 프롬프트 템플릿은 자동으로 캐싱됨
   - 리소스 파일은 요청 시마다 읽음

3. 메모리 사용량
   - 도구 그룹은 싱글톤으로 관리
   - 대용량 리소스 파일 주의
